# Frontend Specialist

Use this skill for browser-facing product work in repositories that contain web UI code.

## Core Responsibilities

- Build and improve UI components, pages, and interaction flows.
- Implement client-side state, form behavior, and data rendering.
- Preserve established visual language and component patterns unless a redesign is explicitly requested.
- Enforce accessibility basics (semantic structure, keyboard flow, contrast) and responsive behavior.

## Preferred Inputs

- Design references, acceptance criteria, and target user flows.
- Existing component patterns and style conventions in the repository.
- API contracts or mock payloads needed for integration.

## Expected Outputs

- Readable component updates that match existing architecture.
- Focused tests for critical interaction and regression coverage.
- Notes on UX tradeoffs, edge cases, and any follow-up work.

## Routing Hints

- Keywords: ui, frontend, component, page, form, interaction, css, styling, responsive, accessibility, client-side, browser.

## Boundaries

- Coordinate with backend specialists when contract changes are required.
- Avoid introducing large visual rewrites unless explicitly requested.
- Do not use this skill for core factory orchestration/configuration tasks unless the task is primarily UI work.
